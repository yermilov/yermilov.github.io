Time to wrap up 2023 and retro things a bit. (Not all the things, it will be too hard, but this blog specifically.) I started it in July, and my main goal was to exercise regularly in writing my thoughts down. I needed a regular source of topics, so I've decided to publish short stories about some particular things I'm working on.

I'm not a big fan of long monumental blogs that almost feel like a monograph and take months to write and hours to read, which was also aligned with this intention. So my usual flow now is to take a topic from the backlog, spend an hour a day for 2-3 days writing most of the material (quite often from the smartphone), and one more hour from the laptop editing (of course using Grammarly) and preparing the final version. The last touch is a thumbnail, which I generate using DALL-E 3 and the prompt that always starts with "Please generate abstract art that resembles something that AI consciousness would generate. I should be smooth and thin colourful waves splitting and merging beautifully. Use pallets of green and one more color of your choice…"

The main result — 12 posts. Today I want to resurface some of them.

### 📈 Most viewed

[Full control over Spotless Java code style](/en/blog/full-control-over-spotless-java-code/) · August 25, 2023

Spotless has become a more and more popular and standard tool for Java code formatting at Grammarly. It is usually trivial to set up, but it has some quite opinionated defaults. I want to share an interesting way we've found to go all the way in the opposite direction and make every possible parameter of code style configurable.

### ❤️ Most liked

[Chatops that was lost and found in time](/en/blog/chatops-that-was-lost-and-found-in/) · September 22, 2023

Automations for development processes unveil hidden complexity in such trivial things you could never imagine there would be any problems. Believe me, today I will talk about a task to print a date and a time.

### 📺 Longest read

[Guiding team processes with ChatOps: On-call](/en/blog/guiding-team-processes-with-chatops/) · December 2023

As your team grows, you need to scale its culture. There is no doubt that things like a high hiring bar are critical, but still, there is a limit to what can be achieved by only smart people talking in the office kitchen and agreeing to do something. Five concrete on-call automations and the lessons each one taught us about scripting team behaviour without losing what made the kitchen-table culture work.
